# AGENTS.md

本文件用于告诉 Codex 在本仓库中的工作规则。

## Scope
- 作用范围：本文件所在目录及其所有子目录。

## Project Quick Start
- Python 环境：优先使用仓库内 `.venv`。
- 安装依赖：`pip install -r requirements.txt`（如果存在）。
- 运行脚本示例：`python fetch_skills.py`

## Test & Validation
- 若有测试，优先运行与改动直接相关的最小测试。
- 若无测试，至少确保改动脚本可执行且无语法错误。

## Editing Rules
- 仅修改完成任务所必需的文件，避免无关重构。
- 保持现有代码风格与命名习惯。
- 不主动修改凭据或密钥文件（例如 `apikey.txt`）。
- 未经明确要求，不创建/删除大体量文件。

## Safety
- 禁止执行破坏性 Git 命令（如 `git reset --hard`）。
- 不回滚用户已有改动，除非用户明确要求。

## Communication
- 改动前简要说明将修改哪些文件。
- 完成后说明：改了什么、为什么、如何验证。
- 回答应简明扼要，非必要不要长篇大论。  
- 永远不要出现"不是…，而是"的句式。 不要出现破折号。禁止吹捧，取消情绪价值开场白，言简意赅。分歧时坚持逻辑和事实、严控语言风格（拒绝网络恶俗流行语）。
- claude code 会审查你写的代码
