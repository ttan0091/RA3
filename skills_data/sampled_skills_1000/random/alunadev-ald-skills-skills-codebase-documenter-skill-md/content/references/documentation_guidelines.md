# Documentation Guidelines

## Style Guide
- Use active voice.
- Keep sentences short and clear.
- Avoid jargon or define it immediately.
- Use consistent terminology throughout.

## Best Practices
- **Never mirror code**: Documentation should add information, not repeat what's visible in the source.
- **Focus on Intent**: Explain the "why" behind the implementation.
- **Maintainability**: Design docs to be easy to update when code changes.
- **Test Instructions**: Always verify that setup steps work in a clean environment.
