# Research Strategies

## Search Query Templates

### For Technical Skills
- `<technology> best practices <year>`
- `<technology> style guide official`
- `<technology> anti-patterns avoid`
- `<technology> performance optimization`

### For Process Skills
- `<process> workflow optimization`
- `<process> automation best practices`
- `<process> efficiency tips`

### For Domain Skills
- `<domain> industry standards`
- `<domain> professional guidelines`
- `<domain> quality checklist`

## Evaluating Sources

**High quality sources:**
- Official documentation
- Reputable tech blogs (e.g., engineering blogs from major companies)
- Academic/research publications
- Industry standard bodies

**Use with caution:**
- Forum answers (verify with multiple sources)
- Outdated articles (check publication date)
- Opinion pieces without evidence

## LLM-Specific Considerations

When improving skills for Claude:

| Aspect | Best Practice |
|--------|---------------|
| Token efficiency | Remove redundant words, use tables over prose |
| Clarity | Imperative voice, concrete examples |
| Structure | Progressive disclosure, overview first |
| Triggers | Comprehensive description with keywords |
| Flexibility | Match specificity to task fragility |

## Common Skill Improvements

1. **Vague triggers** → Add specific keywords and use cases to description
2. **Wall of text** → Break into sections with clear headers
3. **Missing edge cases** → Add troubleshooting section
4. **Outdated practices** → Update with current standards
5. **Overly verbose** → Condense to essential information
