# Design MD & PRD Skill

## The PRD Structure
When asked to plan a feature, generate a Markdown file with these sections:

1. **Problem Statement**: What are we solving and why?
2. **User Stories**: "As a [user], I want to [action], so that [benefit]."
3. **Technical Architecture**:
   - Frontend components needed.
   - Backend API endpoints needed.
   - Database schema changes.
4. **Non-Functional Requirements**: Performance, Security, Reliability.
5. **Milestones**: Step-by-step plan to build it.

## Architecture Diagrams
Use Mermaid.js syntax for diagrams if needed:
- Flowcharts for logic.
- Sequence diagrams for API interactions.
