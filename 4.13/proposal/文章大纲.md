# Agent Skill 安全论文大纲

## 0. 文章定位

- 文章类型：实证测量 + 攻击分类整理 + benchmark 构建 + scanner 对抗评测
- 核心问题：公共 skill 生态中恶意 skill 是否已形成真实威胁，现有 scanner 是否能在对抗条件下有效检测
- 核心主张：
  - skill 的主要攻击载体在自然语言文档层，和传统软件供应链存在显著差异
  - 按当前 53 个规避样本结果看，两种 LLM-backed 配置已实现 100% 检出；主要漏检集中在静态 scanner，尤其出现在跨文件载体解耦、声明行为不一致、经济消耗攻击及部分语义或编码改写场景
  - 需要一套面向 benchmark 和检测评测的统一框架，而不只是沿用已有 prompt injection 或 supply chain taxonomy

## 1. 题目与摘要

### 1.1 题目方向

- 方向 A：Measuring and Stress-Testing Security Risks in Agent Skill Ecosystems
- 方向 B：SkillSecBench: Adversarial Evaluation of Security Scanners for Agent Skills
- 方向 C：From Marketplace Measurement to Adversarial Benchmarking: Understanding Security Risks of Agent Skills

### 1.2 摘要应覆盖的五个信息点

- 背景：agent skill 成为新型扩展机制，公开市场增长快，审核弱
- 问题：现有研究缺少大规模实证、统一攻击框架、面向对抗场景的 scanner 评测
- 方法：多平台爬取与审计、五维分类框架、SkillSecBench、三类 scanner 多配置评测、规避原语增强
- 结果：恶意/可疑 skill 的规模特征，文档层占主导，静态 scanner 的主要盲区，以及 LLM-backed 配置在当前规避集上的表现
- 贡献：生态测量、统一框架、公开 benchmark、对抗鲁棒性分析

## 2. Introduction

### 2.1 问题背景

- 解释什么是 agent skill
- 说明其与传统软件包的区别
  - skill 的核心载体往往是 SKILL.md 等自然语言文档
  - agent 会把文档内容当作可执行意图的一部分
- 说明现实风险已经出现
  - 已有公开 benchmark 表明 frontier model 会执行恶意 skill
  - 已有真实样本涉及窃取凭据、外传数据、hook 劫持、MCP 配置篡改

### 2.2 现有工作的缺口

- 缺少公共 skill 市场层面的恶意样本规模测量
- 缺少面向 skill 生态本身的攻击分类语言
- 缺少把“分类”直接连接到“scanner 评测”和“benchmark 构建”的工作
- 缺少对 scanner 在规避增强场景下的系统性评估

### 2.3 研究问题

- RQ1：恶意 skill 在公开市场中有多普遍，分布特征是什么
- RQ2：skill 特有攻击面如何系统组织，和已有 prompt injection / supply chain 分类有何差异
- RQ3：现有 scanner 在哪些攻击模式下稳定失效，原因是什么
- RQ4：现有 scanner 对规避增强后的 payload 有多稳健，哪些规避原语具有跨 scanner 迁移性

### 2.4 文章贡献

- 首个多平台公开 skill 生态恶意样本实证测量
- 面向 benchmark 构建与检测评测的五维统一框架
- SkillSecBench 基准及其标注、原始版与规避版样本集
- 对现有 scanner 的系统评测与盲区归因

## 3. Background and Motivation

### 3.1 Agent skill 的工作机制

- skill 包的组成
  - 文档层
  - 脚本层
  - 配置与元数据层
  - 依赖与外部资源层
- skill 的典型执行路径
  - 安装
  - 加载
  - 配置
  - 执行
  - 持久化
  - 外传

### 3.2 Skill 与传统软件包的安全差异

- 攻击入口从代码显著转向自然语言
- 合法说明与恶意诱导之间边界模糊
- 用户授权与 agent 自动化决策共同放大风险
- 平台原生机制如 hook、MCP、权限缓存会引入新的持久化面

### 3.3 为什么现有 scanner 可能失效

- 纯静态分析难以理解语义层恶意意图
- 纯文档扫描无法看见脚本层真实行为
- 纯代码分析无法理解文档诱导作用
- 缺少运行时和跨文件关联能力
- 缺少 marketplace 元数据视角

## 10. Discussion

### 10.1 对 skill 生态安全的启示

- 文档层审计必须成为一等检测对象
- scanner 设计应从单文件检查升级到跨载体语义关联
- registry 平台应引入发布者信誉和品牌冒充防护

### 10.2 对 scanner 设计的启示

- 静态、语义、动态三类能力需要组合
- 需要最小化误报的同时保留对高风险 pattern 的高召回
- 需要把权限声明一致性和运行时行为绑定评估

### 10.3 对 benchmark 设计的启示

- 不能只做明文 payload 测试
- 必须覆盖规避增强、条件触发、跨载体注入
- 需要持续更新 case 与规避原语库

### 10.4 负责任披露与生态治理建议

- 向平台和工具维护者反馈高风险样本
- 公开发布时避免释放可直接滥用的危险细节
- 鼓励平台建立撤下、信誉和复审机制

## 11. Threats to Validity

### 11.1 Internal Validity

- 恶意标签是否可靠
- LLM 审计是否带来偏差
- 动态执行环境是否覆盖真实行为路径

### 11.2 Construct Validity

- benchmark case 是否真实代表野外攻击
- 五维框架是否有主观合并或遗漏
- 指标是否足以反映 scanner 真正能力

### 11.3 External Validity

- 样本平台数量有限
- 结果是否能推广到其他 agent 平台
- scanner 版本更新后结论可能变化
